# 高岩财报解读 Skill

> 目标：把“原始财报信息”转成“可决策洞察”  
> 适用对象：餐饮品牌、食品饮料、连锁消费、平台型餐饮相关上市公司

## 必读参考

- 在处理任何餐饮连锁品牌财报前，先读取 [reference.md](reference.md)。
- 其中包含“直营/加盟、系统 GMV/品牌收入、同店、闭店率、汇率、GAAP/Non-GAAP”标准口径与常见误判。
- 需要输出风格对齐时，参考 [examples.md](examples.md) 的两套示例（霸王茶姬/百胜中国）。

## 何时使用

- 用户要求“解读某公司财报/季报/年报”
- 用户要求“提炼核心数据、策略含义、风险点”
- 用户要求“按高岩周报风格输出”
- 用户需要“做成给客户可读的一页解读”

触发关键词示例：
- 财报 / 季报 / 年报 / 业绩会 / earnings / 6-K / 20-F / 10-K / 10-Q
- 核心指标 / 同店 / 门店 / GMV / 利润率 / 现金流 / 指引 / 风险
- 霸王茶姬 / 百胜中国 / 瑞幸 / 海底捞 / 奈雪 / 茶百道 等

## 输入要求（默认自动补全）

最少输入：
- `company`: 公司名
- `period`: 期间（如 2025FY / 2026Q1）
- `source`: 财报来源链接或文件路径

可选输入：
- `peers`: 对标公司列表
- `focus`: 本次关注重点（增长质量/盈利能力/门店效率/海外扩张等）
- `format`: `quick`（5分钟版）/ `standard`（标准版）/ `deep`（深度版）

1) **结构化 MD 解读稿**（可直接入周报）

## 执行流程

### Phase 1｜信息采集（先一手，后二手）

**联网操作须先加载 `/web-access` skill，通过 web-access 执行搜索和页面抓取。**

1. 优先官方与监管来源：
   - 公司 IR（Investor Relations）
   - 监管披露（SEC/HKEX/上交所/深交所）
2. 提取并记录关键表：
   - 损益表（收入、成本、费用、营业利润、净利润）
   - 现金流（经营/投资/融资现金流）
   - 资产负债（现金、债务、租赁负债）
   - 经营指标（门店数、同店、客单、订单、会员、海外）
3. 每个核心数据必须带“来源+口径”。

质量门：
- 若同一指标出现多版本，以“一手披露口径”覆盖媒体口径。

### Phase 2｜指标拆解（先数字，后观点）

至少计算以下指标（缺失则标注“未披露”）：

- 增长类：收入同比、环比（若有）、分业务增速
- 盈利类：毛利率、营业利润率、净利率、非 GAAP 利润率
- 费用类：销售费率、管理费率、研发费率（如适用）
- 现金类：经营现金流/净利润（现金含金量）
- 效率类：单店收入、同店增速、门店净增、闭店率（可得时）

### Phase 3｜高岩解读框架（结论先行）

按以下四问组织观点：

1. **增长是否可持续？**（量价、门店、同店、区域结构）
2. **利润变化来自哪里？**（结构优化还是费用压降）
3. **现金流是否健康？**（利润质量、扩张资金来源）
4. **下一阶段策略重心是什么？**（产品、渠道、组织、海外）

输出时每条观点必须满足：
- 观点句（1句）
- 数据证据（1-2个数字）
- 业务含义（对品牌/赛道意味着什么）

### Phase 4｜策略建议（高岩风格）

给 3 条可执行策略，按“短中长”分层：

- **短期（1-2季度）**：保利润、稳同店、提效率
- **中期（1年）**：结构升级、品类组合、渠道重构
- **长期（1-3年）**：组织能力、供应链、海外与数字化

每条建议用格式：
- `建议`：一句话动作
- `依据`：对应财报数据
- `风险`：执行约束
- `监控指标`：后续追踪 KPI

## 周报排版参考（高岩样式）

对齐 `高岩餐饮周报` 的写作与视觉语气：

- 标题：结论导向 + 期间
- 开头：三句话摘要（不是背景介绍）
- 主体：数据卡 → 结构拆解 → 策略建议
- 文风：短句、强动词、少空话
- 色彩语义（图表）：收入蓝 / 费用灰 / 利润蓝绿

## 快速版（quick）压缩规则

如果用户要快速版，压缩为：

- 3 条结论
- 1 张核心数据表（不超过 8 行）
- 2 条策略建议
- 1 段风险提示

## 质量检查（两阶段，必须全部通过）

### Phase A：脚本硬验证（真实验证，不可跳过）

在写入 MD 文件后，立即运行以下 Python 脚本：

```python
import re, sys

filepath = sys.argv[1]  # 传入生成的 MD 文件路径
with open(filepath, encoding='utf-8') as f:
    content = f.read()

checks = {
    "文件存在且非空":           len(content) > 500,
    "frontmatter 完整（5字段）": all(f in content for f in ["title:", "tags:", "summary:", "links:", "date_processed:"]),
    "含核心数据卡表格":          "| 指标" in content or "| 营收" in content,
    "至少5个可量化数字（含%或亿或万）": len(re.findall(r'\d+\.?\d*\s*[%亿万元]', content)) >= 5,
    "含一手来源链接":            bool(re.search(r'https?://(www\.sec\.gov|ir\.|hkexnews|sse\.com|szse\.cn)', content)),
    "含高岩策略建议章节":        "策略建议" in content or "五、" in content,
    "含数据来源章节":            "数据来源" in content or "一手披露" in content,
}

failed = [k for k, v in checks.items() if not v]
if failed:
    print("❌ 质量门未通过，以下检查失败：")
    for f in failed: print(f"  - {f}")
    sys.exit(1)
else:
    print(f"✅ 质量门全部通过（{len(checks)} 项）")
```
