# Article Writing

> Long-form content that sounds like a person, not a prompt.

## When to activate

Trigger on: blog posts, guides, newsletters, thought-leadership pieces, matching a founder's voice, or restructuring existing copy that reads flat.

## Non-negotiables

1. **Lead with proof** — examples, screenshots, numbers, anecdotes come first; explanation follows.
2. **One idea per section** — if a section tries to say two things, split it.
3. **No hollow openers** — never start with "In today's rapidly evolving landscape" or any variant.
4. **Evidence over adjectives** — replace "game-changer," "cutting-edge," "revolutionary" with the specific thing that changed.
5. **Kill fake vulnerability** — "I'll be honest…" "I have to admit…" are banned unless the admission has real stakes.
6. **No standalone engagement questions** — "But what does this really mean?" followed by the author answering their own question is filler.

## Banned patterns

- "In today's rapidly evolving landscape / world / market"
- "Game-changer" / "cutting-edge" / "revolutionary" / "paradigm shift"
- "Let's dive in" / "Without further ado"
- "At the end of the day"
- Generic transitions: "That said," "Moving on," "With that in mind"
- Adjective stacking without evidence
- Engagement bait questions the author immediately answers

## Process

### 1. Audience lock
Before writing, state in one sentence: who reads this, what they already know, and what they need to walk away with.

### 2. Hard outline
Build a skeleton with:
- Working title (can change later)
- 4-8 section headings that would make sense as standalone tweets
- Under each heading: the ONE proof point or artifact that anchors it

### 3. Draft
- Write each section around its anchor (data, quote, example, screenshot)
- Keep paragraphs ≤ 4 sentences
- Use subheadings every 200-300 words
- End sections with forward momentum, not summary

### 4. Voice pass
If voice samples were provided, rewrite every sentence that doesn't match. Pay attention to:
- Sentence length distribution
- Vocabulary tier (casual vs. technical)
- Humor frequency and type
- How the author handles uncertainty

### 5. Quality gates
- [ ] Every factual claim references a provided source or is flagged as needing one
- [ ] No banned patterns remain
- [ ] Voice matches supplied examples (if any)
- [ ] Opening line would make someone keep reading
- [ ] Closing line is specific, not generic inspiration

## Default voice

If no voice direction is given, default to **sharp operator**: concrete, unsentimental, useful. Reads like someone who has done the thing, not someone who read about it.

## Formatting

- Use **bold** for key terms on first use only
- Use `code` for tools, commands, file names
- Use > blockquotes for direct quotes from sources
- Lists for scannable items; prose for arguments
- No emoji unless the voice sample uses them
