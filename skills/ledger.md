# 杨顺记账 Skill

数据存储路径：`个人知识库/账本/`，命名规则：`{YYYY-MM}.md`

分类体系沿用原账本：
- **支出分类**：餐饮、交通、购物、学习、娱乐、医疗、人情、住房、育儿、商务、其他
- **收入分类**：工资、投资、转账收入、其他收入
- **账户**：微信、支付宝、支付宝花呗、浦发信用卡、招商信用卡、工商银行、其他

---

## 子命令

| 命令 | 用途 |
|------|------|
| `/ledger add` | 记一条或批量记多条 |
| `/ledger import <文件路径>` | 导入历史 Excel/CSV 数据 |
| `/ledger stats [月份]` | 月度/季度汇总报表 |
| `/ledger chart [月份]` | 生成 iOS 风 dashboard 信息图 |
| `/ledger search <关键词>` | 搜索历史记录 |
| `/ledger` | 无参数 → 默认执行 `add` |

## /ledger add — 记账

### 输入格式（结构化）

```
金额 分类 [子分类] [账户] [备注]
```

示例：
```
35 餐饮 午餐 微信 公司楼下
1400 学习 书籍 浦发信用卡 Notion plus
100 交通 打车 支付宝花呗
4580 人情 孝心 工商银行 阿姨护工费
+12000 工资 招商信用卡 3月工资
```

- `+` 开头 = 收入，否则默认支出
- 账户可省略，默认"微信"
- 备注可省略
- 支持一次性粘贴多行批量录入

### AI 解析规则

1. 解析每行输入，识别：金额、交易类型（支出/收入）、分类、子分类、账户、备注
2. 不确定的字段，用括号标注候选，**不要中断录入流程**
3. 解析完毕后展示确认表格：

| # | 类型 | 金额 | 分类 | 子分类 | 账户 | 备注 |
|---|------|------|------|--------|------|------|
| 1 | 支出 | 35 | 餐饮 | 午餐 | 微信 | 公司楼下 |

4. 用户确认后（回复"ok"/"确认"/直接 Enter）写入文件

### 写入格式

读取当月文件 `个人知识库/账本/{YYYY-MM}.md`，若不存在则创建，追加到 `## 记录` 区块末尾：

```markdown
| 2026-04-10 | 支出 | 餐饮 | 午餐 | 35 | 微信 | 公司楼下 |
```

文件结构：
```markdown
---
title: "2026年4月账本"
tags: [账本, 个人财务]
month: 2026-04
---

## 汇总

> 由 /ledger stats 自动更新

## 记录

| 日期 | 类型 | 分类 | 子分类 | 金额(CNY) | 账户 | 备注 |
|------|------|------|--------|-----------|------|------|
```

## /ledger import — 导入历史 Excel/CSV 数据

用法：
```
/ledger import 个人知识库/杨顺的账本WIP.csv
/ledger import 日记/杨顺的记账本2019-2026.xlsx
```

### 原始数据表头（Notion 导出格式）

```
交易类型 | 日期 | 分类 | 分类(1) | 子分类 | 账户1 | 账户2 | 账户币种 | 金额 | 成员 | 商家 | 项目分类 | 备注
```

### 字段映射规则

| 原始字段 | 目标字段 | 处理规则 |
|----------|----------|----------|
| 交易类型 | 类型 | 原样保留（支出/收入/转账） |
| 日期 | 日期 | 统一转换为 `YYYY-MM-DD` 格式 |
| 分类 | 分类 | 原样保留 |
| 子分类 | 子分类 | 去除首尾空格 |
| 账户1 | 账户 | 使用账户1；若为转账则记为"账户1→账户2" |
| 账户币种 | 币种 | 非CNY时保留，CNY时省略 |
| 金额 | 金额(CNY) | 去除逗号，转为纯数字 |
| 商家 | 商家 | 原样保留，空则省略 |
| 项目分类 | 项目 | 原样保留，空则省略 |
| 备注 | 备注 | 合并备注+商家信息 |
| 成员 | （忽略） | 默认本人，不单独存字段 |
| 分类(1) | （忽略） | 重复字段，忽略 |
| 账户2 | 转账目标 | 仅转账类型时使用 |

### 执行步骤

1. **读取文件**：
   - CSV 文件：直接用 Read 工具读取
   - XLSX 文件：用 PowerShell 转换为 CSV 后处理：
     ```powershell
     powershell -Command "[Console]::OutputEncoding = [System.Text.Encoding]::UTF8; Import-Excel '<路径>' | Export-Csv -Path '<输出路径>.csv' -NoTypeInformation -Encoding UTF8"
     ```
     若无 ImportExcel 模块，提示用户先在 Excel 里另存为 CSV。

2. **数据清洗**：
   - 跳过空行和无金额行
   - 日期统一解析（支持 "February 9, 2024 5:06 AM (GMT+8)" 格式）
   - 金额去掉千分位逗号和引号，转为纯数字
   - 子分类去除首尾空格

3. **按月分组**：将所有记录按 `YYYY-MM` 分组

4. **逐月写入**：
   - 检查 `个人知识库/账本/{YYYY-MM}.md` 是否存在
   - 不存在则新建（含标准 frontmatter）
   - 存在则追加（避免重复：按日期+金额+分类去重）
   - 每个文件的 `## 记录` 表格追加对应月份数据

5. **完成报告**：
   ```
   ✅ 导入完成
   共处理：3,368 条记录
   时间跨度：2019-01 ～ 2026-03
   生成月份文件：87 个
   跳过重复：0 条
   跳过无效行：12 条
   ```

### 导入后的标准表格格式

每月文件 `## 记录` 区块内容：

```markdown
| 日期 | 类型 | 分类 | 子分类 | 金额(CNY) | 账户 | 商家 | 项目 | 备注 |
|------|------|------|--------|-----------|------|------|------|------|
| 2024-02-09 | 支出 | 学习 | 书籍 | 1400 | 浦发信用卡 | 其他 | 职业 | Notion plus |
| 2024-02-09 | 支出 | 交通 | 打车 | 100 | 支付宝花呗 | 滴滴 | 日常 | — |
```

## /ledger chart — iOS 风 Dashboard 信息图

用法：
```
/ledger chart          # 本月
/ledger chart 2026-03  # 指定月份
```

### 执行步骤

1. 先执行 `stats` 逻辑，获取汇总数据
2. 将汇总数据整理为结构化 Markdown，包含：
   - KPI 区块：总支出、总收入、净结余（同比上月）
   - 分类支出表格（分类、金额、占比）
   - Top 支出条目列表
   - 月环比变化（若有上月数据）
3. 调用 `/baoyu-infographic`：

```
/baoyu-infographic --layout dashboard --style ui-wireframe --aspect portrait --lang zh
```

**风格说明**：
- `dashboard` 布局：KPI 卡片 + 分类图表 + 排行榜，完美契合账单展示
- `ui-wireframe` 风格：灰度 iOS 界面风，扁平简洁，接近苹果自带钱包/健康 App 视觉
- 竖版（portrait）：适合手机截图分享

4. 生成后输出路径提示

## /ledger search — 搜索

用法：
```
/ledger search 滴滴        # 搜商家
/ledger search 餐饮        # 搜分类
/ledger search 2026-03     # 搜月份
/ledger search >1000       # 搜金额大于1000的记录
```

读取所有 `个人知识库/账本/*.md` 文件，过滤匹配行，表格输出结果 + 合计金额。

## 注意事项

- 禁区：`个人知识库/杨顺的账本WIP/`（Notion 原始导入数据）只读，不要修改
- 新账本存放在 `个人知识库/账本/`（新目录，干净独立）
- 每月文件写入前必须先 Read 确认内容，再 Append，不要覆盖
- 金额单位默认 CNY，外币记录需在备注注明（如：USD 50）
